### 2. 评论权重系统优化

**需求描述**：
- 激烈的评价权重高，平淡的权重低
- 星级评分可以作为权重依据
- 情感分布计算需要加权

**实现要点**：
- [ ] 设计权重计算公式
- [ ] 基于星级的权重：5星/1星权重高，3星权重低
- [ ] 基于点赞数的权重：高点赞评论权重更高
- [ ] 情感分析结果加权计算

**权重计算方案**：
```python
# 方案1：基于星级的权重
星级权重 = |星级 - 3|  # 1星=2, 2星=1, 3星=0, 4星=1, 5星=2

# 方案2：综合权重
综合权重 = 星级权重 * log(点赞数 + 1)

# 方案3：情感强度权重
情感强度 = |情感得分 - 0.5| * 2  # 0-1之间，越接近0或1权重越高
```

**注意**：星级本身已经反映了评价倾向，需要避免重复计算

### 3. 高频词云AI智能筛选

**需求描述**：
- 使用AI筛选词云，仅保留关键词
- 对比AI筛选和正则表达式筛选两种方案

**方案对比**：

#### 方案A：AI筛选（推荐）
**优点**：
- 智能理解语义，筛选更准确
- 可以过滤无意义词汇（如"觉得"、"感觉"）
- 保留真正的关键词（如"救赎"、"希望"、"自由"）

**缺点**：
- 需要调用外部API，有网络延迟
- 有API调用成本（虽然免费额度充足）
- 依赖第三方服务稳定性

**实现方式**：
- 使用智谱AI GLM-4.7-flash模型
- API Key: `f4caddac2c0f4be0bb4acb1a9e9def8b.nDNDOd17sbAQudxK`
- 将高频词列表发送给AI，让其筛选关键词

#### 方案B：正则表达式筛选
**优点**：
- 本地处理，速度快
- 无需网络请求，稳定可靠
- 无额外成本

**缺点**：
- 规则固定，灵活性差
- 难以理解语义，可能误删关键词
- 需要维护大量规则和词库

**实现方式**：
- 扩展停用词库
- 使用正则过滤特定模式（如单字词、数字等）
- 基于词性标注过滤（保留名词、动词、形容词）

**推荐方案**：AI筛选（方案A）
- 效果更好，用户体验更佳
- 可以在AI不可用时降级到正则筛选

## 🐛 已解决的问题

### 问题1：豆瓣反爬虫限制
**现象**：
- 网页版返回418/403状态码
- 返回验证页面，无法解析数据

**解决方案**：
- 改用豆瓣移动端API
- 使用移动端User-Agent
- 无需Cookie即可访问

### 问题2：搜索API返回空数据
**现象**：
- 搜索接口返回418状态码
- JSON解析失败

**解决方案**：
- 从JSON API改为移动端搜索API
- 使用 `m.douban.com/rexxar/api/v2/search/movie`

### 问题3：评论采集失败
**现象**：
- 提示"需要登录Cookie"
- 采集0条评论

**解决方案**：
- 使用移动端评论API
- 端点：`m.douban.com/rexxar/api/v2/movie/{id}/interests`

### 问题4：NoneType错误
**现象**：
```
AttributeError: 'NoneType' object has no attribute 'get'
```

**原因**：
- 某些评论的 `rating` 字段为 `None`（用户未打分）
- 某些评论的 `user` 字段为 `None`

**解决方案**：
- 增加空值检查
- 为None值设置默认值（星级=0，用户名="匿名"）

### 问题5：电影信息获取失败
**现象**：
- 电影详情页返回验证页面
- 无法解析电影名称和评分

**解决方案**：
- 改用移动端电影详情API
- 端点：`m.douban.com/rexxar/api/v2/movie/{id}`
